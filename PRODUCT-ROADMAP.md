# Tripboard 功能大纲

这个文档用于防止后续迭代遗忘和跑偏。每完成一项，就在这里同步勾选并补充状态。

## 状态说明

- `[x]` 已完成并已推送公网
- `[ ]` 未完成
- `[~]` 部分完成，仍有后续依赖或升级项

## 功能清单

### 1. 交通票价

- [~] 官方查询入口
  - 已完成：携程查询、Trip.com 查询、12306 查询入口。
  - 已完成：根据当前出发地、目的地、日期生成查询关键词。
- [~] 手动保存报价
  - 已完成：可手动保存航班/动车班次、时间、价格到当天交通列表。
  - 已完成：手动报价会进入协作计划并同步。
- [~] Supabase 交通代理
  - 已完成：前端可调用 Supabase Edge Function。
  - 当前限制：无正式 Trip.com/Ctrip API 权限时只返回代理示例。
- [~] Amadeus 个人开发者航班 API
  - 已完成：新增 Supabase Edge Function `amadeus-flight-offers`，用于安全调用 Amadeus OAuth 和 Flight Offers Search。
  - 已完成：前端接入面板已改为 Amadeus 航班报价测试，支持国内航线的城市/机场三字码输入。
  - 当前状态：已部署到 Supabase，但 Amadeus Self-Service 新注册入口已不可用；仅旧账号可能继续使用。
  - 当前限制：Amadeus Self-Service portal 已提示将于 2026-07-17 下线，不再作为主线方案。
  - 当前限制：Amadeus 只解决机票报价；动车/高铁仍需 12306 查询、手动保存，或后续接入有授权的火车票 API。
- [~] SearchApi / Google Flights 航班 API
  - 已完成：新增 Supabase Edge Function `searchapi-flight-offers`，用于安全调用 SearchApi 的 Google Flights API。
  - 已完成：前端接入面板改为 Google Flights 报价接入，默认代理地址切到 `searchapi-flight-offers`。
  - 已完成：`SEARCHAPI_API_KEY` 已配置到 Supabase secrets。
  - 已完成：已用 `SHA -> LHW`、`2026-07-10` 测试国内航线，成功返回 10 条 Google Flights 航班报价。
  - 当前限制：这是聚合搜索接口，不是航空公司或 12306 官方接口；适合先做航班价格可视化验证。
- [~] 多出发地综合比较
  - 已完成：可填写多位成员的出发地/机场码，例如 `林: SHA; 王: PEK; 周: CAN`。
  - 已完成：按同一到达地、同一日期和时间筛选逐个查询 Google Flights / SearchApi，并展示每人的最低价、航班、耗时、经停和推荐等级。
  - 当前限制：每个出发地会消耗一次 SearchApi 查询；当前最多比较 6 个出发地。
  - 当前限制：暂时只覆盖航班；动车/高铁仍需要 12306 查询或手动保存。
- [ ] 真实 Trip.com/Ctrip 机票/动车 API
  - 未完成原因：需要具体产品 API 权限、AppKey/AppSecret、接口文档。
  - 当前判断：个人账号大概率无法直接申请完整机票/火车票 API。

  - 已完成：前端会把日期、节奏、预算、天气、地点类型、备注、标签、经纬度一起传给代理。

  - 已完成：TokenFlux 新建 `tripboard-supabase` 专用 key，使用 `阿里云百炼（OpenAI格式）` 分组，避免旧 Codex 专用 key 的 TLS 路由限制。

  - 已完成：当前使用 `https://tokenflux.dev/v1` + `gpt-5.5`，后端已实测返回 `OpenAI compatible · gpt-5.5`。
  - 已完成：最新版 `ai-route-optimize` 已部署到 Supabase，并已实测返回 `order`、`note`、`reason`、`segments`、`warnings`。
  - 已完成：后端会用真实地点标题修正 `segments.from/to`，避免模型或请求编码导致分段地点名显示异常。
  - 已完成：模型提示词已要求考虑地理位置、固定时间、餐点、预约/开放提示、天气、折返、体力、预算和标签。
  - 待实现：自动接入营业时间、实时交通、排队数据和景区拥挤度；目前这些仍依赖地点备注/标签或后续 API。

  - 已完成：可选择类别、日期，填写名称、时间、金额、已付、付款人、地址、订单号、来源链接和备注。
- [x] AI 辅助文本解析
  - 已完成：新增并部署 Edge Function `external-order-parse`。
  - 已完成：前端默认接入 Supabase 解析代理，粘贴订单短信、邮件、分享文本后可解析名称、日期、时间、金额、已付、地址、订单号和备注。
  - 已完成：解析结果会先显示预览，再由用户确认导入。

- [x] 导入到协作计划
  - 已完成：确认导入后会新增到匹配日期；订单日期不在行程范围内时导入当前天并在预览里提示。
  - 已完成：导入记录会写入预算、已付、付款人、来源、类别、订单号、解析来源和原文，并通过 Supabase 同步给协作者。
- [ ] 自动读取外部 App 订单
  - 未完成：需要平台授权/API。
  - 当前限制：不能直接读取用户手机 App 或账号订单。
  - 当前判断：个人网页不能绕过美团/点评、酒店/民宿、OTA 平台授权；后续只能接官方开放接口、企业接口，或继续使用用户主动粘贴/分享导入。

  - 已完成：基于 Supabase Presence 显示协作者正在编辑的文本字段、光标位置和选区长度；字段旁会出现协作者提示。

  - 已完成：新增、删除、上移/下移行程日会实时广播给协作者；当天日期、标题、路线、天气、交通字段可直接编辑并同步。

## 后续优先级建议

1. 先完成 AA 结算/转账建议，因为不依赖第三方平台授权。
2. 再做成员权限分级，包括登录、管理员、成员、访客和邀请审批。
3. 再接高德地图 SDK，因为能显著提升路线和定位体验。
4. 继续扩展 Notion 式协作，从地点文本字段组升级到多人光标、选区和更多结构字段的 CRDT 合并。
5. 真实携程/Trip.com 票价 API 等拿到企业权限后再接。
